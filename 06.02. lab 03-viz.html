Deployment Phases

Pre-flight Checklist

Verify these before installing anything. Capture a screenshot of your laptop's About/System info panel.

  • CPU supports virtualisation (Intel VT-x or AMD-V). Check via systeminfo | findstr Hyper-V (Windows) or lscpu | grep Virtualization (Linux).
  • At least 8 GB RAM (16 GB recommended).
  • At least 60 GB free disk.
  • Virtualisation enabled in BIOS or UEFI.
  • On Windows 10/11: Hyper-V disabled (VirtualBox conflicts with it).

Install VirtualBox & Network

  1. Download VirtualBox 7 and install with default selections.
  2. Install the matching Extension Pack (adds USB pass-through, RDP, PXE).
  3. Create a Host-Only Network: File → Tools → Network Manager → Host-only Networks → Create.
  4. Name: vboxnet-cyb01
  5. IPv4: 10.10.10.1/24
  6. Disable DHCP (We need predictable IPs).

Download and Prepare VMs

Download images from official sources. Verify SHA-256 and check URLs on VirusTotal.

Install GNS3

  1. Download GNS3 all-in-one installer. Accept GNS3 VM option.
  2. Import GNS3 OVA into VirtualBox. Start it and note the IP.
  3. In GNS3 App: Edit → Preferences → GNS3 VM. Enable and point to the VM IP.
  4. Edit → Preferences → VirtualBox VMs → New. Select cyb01-opnsense.
  5. Build a test topology: Ethernet switch + OPNsense VM + VPCS host. Verify ping from VPCS to OPNsense LAN.

AWS Academy Access

  1. Access AWS Academy Learner Lab via Canvas module.
  2. Start Lab, click AWS icon to open Management Console.
  3. Launch t2.micro Ubuntu EC2 (us-east-1, default VPC). Security group: SSH from your home IP only.
  4. Generate key pair, download .pem.
  5. Connect via SSH.

Day 06: SOC Stack Integration

Wireshark, SNMP, Wazuh, and Splunk with OPNsense.

You have the plumbing; now we install the brain. By the end of this lab, you will have a functional SIEM workflow capturing live network traffic, receiving syslogs, and alerting on Windows endpoint security events inside a unified Splunk dashboard. Ensure all VMs are at the 01-networked snapshot before proceeding.

Phase 1: Wireshark on Capture VM

Use the Ubuntu VM as the dedicated capture host. Mirror the OPNsense LAN switch traffic to it via GNS3 (right-click switch port → Configure → Port Mirroring).

# Install Wireshark + tshark sudo apt update && sudo apt install -y wireshark tshark # Add user to wireshark group (avoid root capture) sudo usermod -aG wireshark $USER newgrp wireshark # Start a 30-second capture (verify interface name with ip link) sudo tshark -i enp0s3 -a duration:30 -w /tmp/cyb01-baseline.pcapng

Action: While capturing, generate traffic from the Windows VM (browse, ping). Open the pcapng in Wireshark on your host and filter by http or dns or icmp.

Phase 2: SNMP on OPNsense

Enable SNMP in OPNsense GUI (Services → SNMP). Set community string to labdemo.

# On Ubuntu VM: Install Net-SNMP tools and MIBs sudo apt install -y snmp snmp-mibs-downloader sudo download-mibs # Walk the system tree snmpwalk -v2c -c labdemo 10.10.10.1 1.3.6.1.2.1.1

Reflection Prompt Requirement

Run a Wireshark capture filtered on snmp during the walk. Notice the community string in cleartext. In your report, discuss SNMPv3 (RFC 3414) vs v1/v2c.

Phase 3: Install Wazuh Single-Node

⚠️ Pre-requisite: Shut down Ubuntu VM and increase RAM to 4GB before running this script.

# Download official installation assistant curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh # Run in all-in-one mode sudo bash ./wazuh-install.sh -a

Access dashboard at https://10.10.10.30. Save generated admin credentials immediately.

Generate an Alert (Windows VM)

Enrol the Windows agent via the dashboard wizard. Then, force authentication failures via PowerShell as Admin:

1..5 | ForEach-Object { Start-Process -FilePath "runas.exe" ` -ArgumentList "/user:nonsense$_ cmd.exe" ` -ErrorAction SilentlyContinue }

Phase 4: Install Splunk Free

Splunk will coexist on the 4GB Ubuntu VM. Download the .deb from Splunk manually (account required).

# Install and configure Splunk sudo dpkg -i splunk-9.x.x-amd64.deb sudo /opt/splunk/bin/splunk start --accept-license sudo /opt/splunk/bin/splunk enable boot-start
  • Access via http://10.10.10.30:8000.
  • Switch to Free License in Settings.
  • Enable HTTP Event Collector (HEC) and create token wazuh-feed.

Forward Wazuh Alerts

Edit /var/ossec/etc/ossec.conf on Ubuntu to add the custom-splunk integration block, using your HEC token. Restart wazuh-manager.

Phase 5: Cross-OS Log Exploration

A SIEM abstracts logs, but you must know where they originate. Trigger another Windows failed logon and track it.

🐧 Linux (Manager)

sudo tail -f /var/ossec/logs/alerts/alerts.log

Watch alert decisions live.

🪟 Windows (Agent)

C:\Program Files (x86)\ossec-agent\ossec.log

Find the receipt confirmation line.

🧱 OPNsense (FW)

Add syslog target: 10.10.10.30:514 UDP.

sudo tcpdump -i any -n udp port 514

Run on Ubuntu to see incoming firewall logs.
